🟠 Backblaze B2 — Storage de vídeos (~$1/mês para 180GB)
Como configurar:
1. Crie conta em backblaze.com (gratuito até 10GB, depois $0,006/GB)
2. Buckets → Create a Bucket → marque Public → Create Bucket
3. App Keys → Add a New Application Key → copie keyID e applicationKey
4. No bucket: clique em Endpoint para ver o endereço S3
No bucket → "Bucket Settings" → "Friendly URL". Formato: https://f00X.backblazeb2.com/file/NOME-DO-BUCKET
☁️ Deploy no Railway — rodar sem PC ligado
💰 Custo mensal
Railway: $5/mês
Backblaze 180GB: ~$1/mês
Total: ~$6/mês
📊 Capacidade
60+ contas ✅
12.000+ posts (SQLite) ✅
180GB+ vídeos (B2) ✅
# 1. Suba o projeto no GitHub (github.com → New repository)
# 2. Deploy no Railway
railway.app → New Project → Deploy from GitHub repo
# 3. Variáveis de ambiente (aba Variables no Railway):
PORT=3000
B2_KEY_ID=sua_key_id
B2_APP_KEY=sua_application_key
B2_BUCKET=nome-do-bucket
B2_ENDPOINT=https://s3.us-west-004.backblazeb2.com
B2_PUBLIC_URL=https://f004.backblazeb2.com/file/nome-do-bucket
# 4. Settings → Networking → Generate Domain → acesse de qualquer lugar!
⚠️ Atenção SQLite + Railway: O Railway usa disco efêmero — o db.sqlite pode ser apagado ao reiniciar. Para produção com 60 contas, recomendo adicionar um volume persistente no Railway (Storage → Add Volume → montar em /app/data).
